The issue you're facing is likely due to negative caching, where the CDN caches the 404 Not Found response for your new files. When you first tried to access them, they didn't exist in the cache, the CDN checked your server, got a "nope," and then saved that "nope" response for a while.

## 1. Immediate Solution: Purge the CDN Cache
This is the fastest and most likely solution. It tells Hostinger's CDN to forget everything it has saved for your site and fetch fresh copies.

Log in to your Hostinger hPanel.

Navigate to the Performance section in the sidebar.

Click on CDN.

You should see a button labeled Purge All or Flush Cache. Click it.

Wait a few minutes for the purge to propagate across the CDN network. Then, try accessing your new webhook.php file again.

This single step resolves the problem for most users.

## 2. Workaround: Use a Query String (Cache-Busting)
If purging the cache is slow or doesn't work, you can trick the CDN into thinking it's a completely new URL by adding a meaningless query string to the end. The CDN will see a new URL, bypass its cache, and request the file from your server.

Update your Google Apps Script to call the webhook URL like this:

https://dashboard.giftedworld.org/webhook.php?v=1

Or, even better, use a timestamp to guarantee it's always unique:

https://dashboard.giftedworld.org/webhook.php?t= + new Date().getTime();

Your PHP script at webhook.php will simply ignore the extra ?t=... part, but for the CDN, it's a brand new link that it has to fetch from the source.

## 3. Long-Term Fix: Disable Cache for Webhooks
Webhooks, by nature, should never be cached. You want them to execute in real-time every time they are called. You can instruct browsers and CDNs not to cache specific files by adding rules to your .htaccess file.

Add the following code to the top of your .htaccess file in the dashboard.giftedworld.org directory:

Apache

# Do not cache webhook files
<FilesMatch "^(webhook\.php|form_webhook_2025\.php)$">
    Header set Cache-Control "no-cache, no-store, must-revalidate"
    Header set Pragma "no-cache"
    Header set Expires 0
</FilesMatch>
What this does:

FilesMatch: This targets only your specific webhook files.

Cache-Control: The modern, primary instruction telling all caches (CDN, browser, etc.) not to store this file at all.

Pragma & Expires: These are legacy headers for older systems, providing backward compatibility.

After adding this, you may still need to do one final cache purge (Step 1) to make the CDN aware of the new .htaccess rule. From then on, it should never cache those files again.y
