Below: immediate today workarounds (fast → fastest), then permanent fixes & diagnostic steps.

Immediate (do now) — zero DNS changes

Fastest, reliable: route the webhook through an existing working PHP file (POSTs generally bypass CDN cache and are forwarded to origin).

Edit an already-working file you control (e.g. leads.php) and add a tiny dispatcher that validates a secret and 
includes/executes your webhook.php on the origin. Because the request is a POST, CDNs normally pass it to origin 
rather than serve a stale cached GET/404. Stack Overflow +1

// paste into leads.php (near top). Use a strong secret; keep it private.
if ($_SERVER['REQUEST_METHOD']==='POST' && isset($_GET['x']) && hash_equals($_GET['x'], 'PUT_A_LONG_SECRET_HERE')) {
    // normalize so included script sees expected content
    $_SERVER['REQUEST_URI'] = '/webhook.php';
    $raw = file_get_contents('php://input');
    $ct  = $_SERVER['CONTENT_TYPE'] ?? '';
    if (stripos($ct,'application/json') !== false) {
        $_POST = json_decode($raw, true) ?? [];
    } elseif (stripos($ct,'application/x-www-form-urlencoded') !== false) {
        parse_str($raw, $_POST);
    }
    // run webhook logic from file on origin
    include __DIR__.'/webhook.php';
    exit;
}
Point Google Apps Script to https://dashboard.giftedworld.org/leads.php?x=PUT_A_LONG_SECRET_HERE.

Why this works: POST often gets forwarded to origin (no edge-GET cache) so your include executes on the server and reads the real webhook.php file.

Immediate (hPanel actions — if you can log in)

Purge Hostinger CDN cache (specific URL or full purge) — hPanel → Websites → Manage → Cache Manager / CDN → Purge (specific URL or Purge all). Hostinger also has a “Development mode” you can enable to bypass CDN for quick testing. This forces edges to re-pull origin copies.

Quick headers & .htaccess tweaks (helpful)

Add response headers at the top of webhook code (or via .htaccess) to prevent caching while testing:

header('Cache-Control: no-store, no-cache, must-revalidate, max-age=0');
header('Expires: 0');
